## 项目背景（供 Codex 理解上下文）

### 核心概念

走马村是一个真实的中国乡村。项目把"认养一棵荔枝树"作为用户与村庄建立长期关系的起点——用户选择一棵树、签订认养协议、追踪树的成长、参与养护活动、最终收获果实。

从这一棵树出发，系统串联起了走马村的四个空间境界：
- **古道叙事境** — 古驿道、历史路径、文化导览
- **荔田共生境** — 荔枝园、认养、采摘、伴手礼（认养入口所在地）
- **韧谷研学境** — 水系、研学课程、安全教育
- **岭上共居境** — 院落民宿、村宴、晨间农事体验

### 产品逻辑

```
认养一棵荔枝树（建立关系）
    ↓
树档案 + 成长记录 + 养护参与（长期陪伴）
    ↓
游客由此了解四境、预约院落、购买门票（转化交易）
    ↓
村民协作者接收任务、更新库存、处理反馈（运营闭环）
    ↓
AIGC 云脑生成路线、导览摘要、天气联动（智能增强）
```

### 技术事实

- **Monorepo**：`apps/web`（前台 PWA）、`apps/admin`（运营后台）、`packages/*`（共享包）
- **技术栈**：Next.js 14 + TypeScript + Prisma + PostgreSQL
- **特色模块**：荔枝树认养流程、认养履约规则模拟引擎、AIGC 路线生成
- **模拟系统**：独立的 `@zouma/simulation` 包，用于 Demo 展示和认养规则验证（不写入真实订单）
- **当前状态**：MVP 已实现前后台核心链路，包括四境内容、认养流程、院落预约、门票预购、支付演示

### 视觉风格

克制、空间化、可信。低饱和、有层次、有乡村真实感。设计原则见 [产品定位](../docs/product/PRODUCT_POSITIONING.md)。
