## Slide 2: 项目定位

**一句话**：以 AIGC 云脑为核心的乡村数字化运营平台

**核心命题**：
> 走马村是重庆长寿区一座普通的山地村落——没有著名景点，荔枝也不是唯一优势，古道、水系、院落各自孤立。**如何让一座普通村庄被看见、被参与、被持续运营？**

**三大目标**：
| # | 目标 | 手段 |
|---|------|------|
| 1 | 前置传播 | AIGC 生成内容，让人先在云端产生兴趣 |
| 2 | 到访入口 | 认养、预约、路线生成，把"逛一下"变成"参与一次" |
| 3 | 持续运营 | 通知推送、互动任务、数据反馈，让村庄持续迭代 |

## Slide 3: 不走的路（反模式）

**我们明确拒绝的做法**：
- ❌ 不做普通乡村旅游门户（堆图片+电话）
- ❌ 不做炫技型 AI SaaS 首屏（蓝紫霓虹控制台）
- ❌ 不做景区复制（没有著名景点也能运营）
- ❌ 不做"只有口号没有运营逻辑"的展示站

**我们的选择**：
- ✅ 以"认养一棵树"建立长期关系
- ✅ 以真实乡村资源为核心
- ✅ 前台像建筑竞赛展板 + 数字产品，后台像稳定控制台
- ✅ 空间化设计：视觉层次来自地形、路径、节点、院落

## Slide 5: 四境空间模型

**系统的核心空间组织逻辑**：

| 境 | 场景 | 核心体验 | 数字能力 |
|---|---|---|---|

## Slide 7: AIGC 路线生成

**用户输入** → **AI 推理** → **路线输出**

**输入维度**：
- ⏱ 时长（半日/一日/两日）
- 👥 人群（老人/亲子/普通/团建）
- 🌤 天气偏好（晴天优先/雨天备选）
- 🚶 体力等级（轻松/中等/挑战）

**输出路线卡包含**：
- 总时长 + 步行强度 + 适合人群标签
- 雨天备选方案
- 需提前预约节点高亮
- 高德地图路线 + 天气校验

**技术实现**：
- 路线模板匹配引擎（`route_template` 表）
- 和风天气 API 实时校验
- DeepSeek 模型生成导览词
- 3 秒内返回首屏结果

## Slide 8: 认养系统 — 一棵树带动全村运营

**认养流程**：
```
浏览树档案 → 选择荔枝树 → 签署认养协议 → 支付 → 激活互动任务 → 持续养护参与 → 收获兑现
```

**核心功能**：
| 功能 | 说明 |
|------|------|
| 🌳 树档案 | 树种、树龄、健康状况、位置（脱敏）、照片 |
| 📋 认养协议 | 在线签署、权益说明、收获分配 |
| 📝 养护日志 | 浇水/施肥/拍照/日记上传 |
| 🎯 互动任务 | 自动生成 4 次浇水 + 1 次施肥 + 1 次拍照 + 1 次日记 |
| 🏆 认养证书 | 电子证书生成 + 成长记录时间线 |
| 📦 收获物流 | 采摘预约 + 物流发货 |

**设计理念**：从"一次性门票"变成"年周期关系"，让认养者成为村庄的长期参与者。

## Slide 9: 院落预约与票务

**院落预约**：
- 日历库存组件：可订 / 已订 / 维护 三态

- 定金/全款灵活选择
- 入住码生成 + 入住须知展示

**门票系统**：
- 三种产品类型：景点票 / 活动票 / 套餐票
- 日期 + 数量选择
- 核销码生成与验证
- 退款流程（原路退回）

## Slide 13: AI 模型编排层

**多模型智能路由**：

| 模型 | 用途 | 优势 |
|------|------|------|
| **DeepSeek-Chat** | 路线生成、导览词、反馈总结 | 低成本推理（默认） |
| **OpenAI** | 高质量创作、多模态 | 内容品质 |
| **Claude** | strict schema、工具调用 | 结构化输出 |
| **Qwen/百炼** | 国内替代、高并发 | 合规 + 弹性 |

**编排规则**：
- 路由选择基于：任务类型 + 成本预算 + 延迟要求 + 合规
- 所有调用记录：provider, model, tokens_in, tokens_out, cost, latency_ms
- 高频场景优先使用 mini 或国产模型
- 禁止每个用户请求触发 Web Search

**AIGC 应用场景**：
1. AI 路线生成 + 导览词
2. AI 日报自动生成（每日 09:00 前）
3. AI 养护建议推送
4. 反馈工单智能分类与摘要
5. 场景内容 AI 讲解摘要

## 附录：PPT 制作建议

### 视觉风格
- **色调**：低饱和、大地色系（米白、灰绿、赭石、深棕）
- **字体**：标题用衬线体（如思源宋体），正文用无衬线体（如思源黑体）
- **图片**：优先使用走马村实地照片（`走马岭实地照片/` 目录），抽象图形仅用于系统关系解释
- **图标**：线性图标，保持克制
- **动效**：淡入淡出，不使用花哨转场

### 每页建议元素
- 左侧或顶部：章节导航标识（当前所处章节高亮）
- 右下角：页码
- 图表：使用 Mermaid 风格或简洁线框图
- 代码：仅在必要时展示，使用暗色背景 + 高亮关键行
