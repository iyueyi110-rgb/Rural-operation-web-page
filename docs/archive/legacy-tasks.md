# 走马村前台系统 — 任务拆解与验收标准

> 基于第一份执行计划（前端网页端与AIGC场景体验系统），将项目拆解为11个Epic，明确优先级、目录归属、首个PR内容和完成定义。

---

## 1. 开发优先级总览

### 优先级分级

| 优先级 | 标记 | 说明 |
|--------|------|------|
| **P0 - 必须** | 🔴 | MVP核心，必须第一版上线 |
| **P1 - 重要** | 🟡 | MVP增强，尽量第一版包含 |
| **P2 - 架构预留** | 🟢 | 架构支持，后续实装 |
| **P3 - 后续** | ⚪ | 明确不在MVP范围 |

### MVP 范围判断

> MVP 必须压缩在"能成交、能导览、能积累数据"的范围里，不建议一开始就上 AR、数字人、复杂 3D 漫游。

---

## 2. 任务拆解（Epic → PR）

### Epic 1: 前台底座 🔴 P0

| 维度 | 内容 |
|------|------|
| **建议目录** | `apps/web`, `packages/ui`, `packages/contracts` |
| **首个PR内容** | 初始化 Monorepo、设计令牌、路由骨架、环境变量 |
| **依赖** | 无 |

**子任务**：
- [ ] 初始化 Monorepo 结构（Turborepo / Nx）
- [ ] 配置 Next.js App Router 项目（`apps/web`）
- [ ] 建立共享 UI 组件库（`packages/ui`）
- [ ] 建立 API 合约包（`packages/contracts` — OpenAPI 类型生成）
- [ ] 配置 ESLint、Prettier、TypeScript strict mode
- [ ] 配置环境变量管理与 .env 模板
- [ ] 搭建 CI/CD 流水线（GitHub Actions / GitLab CI）

**完成定义**：本地可跑 (`npm run dev`)、CI 通过、基础页面可访问（`/` 返回 200）

**完成定义**：3秒内返回首屏结果；能区分老人/亲子/雨天三类样例路线并可回放测试

## 3. 每条任务验收标准汇总

| 验收域 | 验收标准 | 关联 Epic |
|--------|----------|-----------|
| **四境内容** | 所有场景页均有结构化内容、更新时间、封面图、CTA 按钮 | Epic 3 |
| **路线生成** | 3秒内返回首屏结果；能区分老人/亲子/雨天路线；可回放测试 | Epic 4 |
| **天气联动** | 遇预警或降雨自动切换为低风险路线或弹窗提示 | Epic 4 |
| **院落预约** | 不超卖；库存锁定与支付超时释放可复现；日历状态准确 | Epic 5 |
| **票务下单** | 下单、支付、退款、核销全流程可测；订单中心闭环 | Epic 6 |
| **认养功能** | 树档案完整；认养关系可查；日志上传正常；证书生成闭环可跑通 | Epic 7 |
| **支付编排** | 微信/支付宝至少接通一条沙箱流程；回调验签正确；幂等处理有效 | Epic 8 |
| **反馈工单** | 提交、分类、分派、关闭、评价状态完整流转 | Epic 9 |
| **权限安全** | 关键后台菜单按角色隔离；支付回调与退款操作可审计 | Epic 10 |
| **多语言** | 至少做到键值级国际化，不出现混合 hardcode 文案 | Epic 2 |
| **性能** | 首页 LCP < 2.5s、移动端首屏 < 3s、弱网体验可接受 | Epic 11 |
| **内容可信** | 历史/文化类 AI 输出必须带来源与更新时间，不允许无依据发挥 | Epic 3, 4 |
| **CI/CD** | 本地可跑、CI 通过、基础页面可访问 | Epic 1 |

## 4. Epic 依赖关系与建议执行顺序

```
Phase 1 (Week 1-2)     Phase 2 (Week 3-6)       Phase 3 (Week 7-12)     Phase 4 (Week 13+)
─────────────────────   ──────────────────────   ─────────────────────   ─────────────────
Epic 1: 前台底座 ───┬──→ Epic 3: 四境内容 ────→ Epic 11: 试运营埋点
                    │
Epic 2: 国际化SEO ──┤
                    │
Epic 10: 隐私权限 ──┼──→ Epic 5: 院落预约 ──┬─→
                    │                        │
Epic 8: 支付编排 ───┘                        │
                                             │
                    ┌────────────────────────┘
                    │
                    ├──→ Epic 6: 门票套餐
                    ├──→ Epic 7: 认养养护
                    └──→ Epic 9: 反馈工单

Epic 4: 路线生成（依赖 Epic 3 完成后启动）
```

### 并行开发建议

| 阶段 | 可并行的 Epic |
|------|--------------|
| **Phase 1** | Epic 1 + Epic 2 + Epic 10 可并行（共享底座 + 权限模型可同时建立） |
| **Phase 2** | Epic 3 + Epic 5 + Epic 8 可并行（内容、预约、支付三条独立业务线） |
| **Phase 3** | Epic 4 + Epic 6 + Epic 7 + Epic 9 可并行（依赖底座就绪后各自独立开发） |
| **Phase 4** | Epic 11（依赖前面所有Epic完成后引入埋点与性能优化） |

## 6. 不在 MVP 范围内的功能（防范围蔓延）

| 功能 | 何时引入 | 说明 |
|------|----------|------|
| AI 数字人 / 沉浸视频 | 第二阶段 | 在内容与交易闭环跑通后再做 |
| 会员等级/积分 | 第二阶段 | 适合复购期再引入 |
| 社区话题/UGC | 第二阶段 | 先做轻反馈，不做重社区 |
| 3D/AR 漫游 | 第二阶段 | MVP 为"叙事页 + 音频 + 交互卡片" |
| 实时流式分析 | 第二阶段 | 初期日更够用 |
| IoT 生态感知接入 | 第三阶段 | 水位、水质、果园传感器 |
| 自动执行动作 | 后续 | 先建议，不自动下发生产操作 |
| 动态定价 | 后续 | 在经营数据充足后再做 |

---

> **核心提醒**：先锁定 MVP，只保留能形成转化闭环的功能。避免同时做 App、AR、数字人、商城、会员、IoT 导致范围失控。
