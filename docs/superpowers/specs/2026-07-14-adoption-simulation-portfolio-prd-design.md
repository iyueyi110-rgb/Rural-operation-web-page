# “认养一棵树”多场景规则测评系统作品集 PRD 设计

## 1. 文档目标

为“认养一棵树”V0/V1 多场景规则模拟工作台生成一份作品集导向的 PRD 流程文档。成稿主要面向产品负责人、面试官与业务评审者，重点展示产品逻辑、业务闭环、规则设计、异常处理和决策方法，同时保留足够的需求与验收信息，使文档具备正式评审价值。

## 2. 产品定位

系统定位为“认养履约业务的多场景规则测评工作台”。它在固定、可重复的模拟世界中，对认养订单、任务生成、村民分配、接单、执行、提交、审核、退回、权益履约和结算进行全链路预演，并在相同外部条件下比较 V0 与 V1 的规则效果。

核心价值是：在真实运营规则上线前，用一致的数据口径识别流程瓶颈、异常风险和场景退化，形成可追溯的策略升级依据。

所有模拟结果必须持续说明：**模拟运营数据，不代表真实业务结果。**

## 3. 叙事主线

成稿围绕以下业务闭环展开：

```text
创建认养订单
→ 支付并激活认养
→ 按农事周期生成任务
→ V0/V1 规则分配执行者
→ 村民接单并执行
→ 提交图文凭证
→ 规则预检查
→ 人工审核或退回重提
→ 完成权益履约
→ 汇总事件、指标与 Bad Case
→ 判断是否建议升级 V1
```

叙事重点不是技术架构，而是产品经理如何定义问题、拆解角色、约束规则、覆盖异常、统一指标并形成决策闭环。

## 4. 内容结构

成稿使用以下固定章节：

1. 项目摘要与一句话价值
2. 业务背景、用户痛点与产品目标
3. 产品范围及非目标
4. 角色、权限与责任链
5. 端到端业务流程图
6. 认养、任务、审核、权益状态机
7. 核心功能需求与异常流程
8. V0/V1 规则差异及产品决策依据
9. 八类压力测评场景
10. 指标口径、升级门槛与结果解释
11. Bad Case 复盘流程
12. 页面信息架构与操作路径
13. 验收标准和数据真实性边界
14. 产品经理复盘：决策、取舍、局限与后续迭代

## 5. 角色与责任设计

PRD 覆盖五类角色：

- 认养用户：完成认养并查看履约结果，不直接参与模拟管理。
- 运营人员：配置场景、发起测评、处理升级人工的异常、查看指标并形成策略结论。
- 村民执行者：接单、执行农事任务、提交结构化凭证并处理退回。
- 审核员：审核凭证、退回不合格提交并保障关键权益任务优先处理。
- 规则系统：生成任务、执行分配与提醒、进行规则预检查、记录事件、计算指标和输出建议。

每个角色在成稿中都需要明确目标、输入、动作、输出和责任边界。

## 6. 功能需求表达方式

每项核心功能统一使用以下模板：

- 功能目标
- 使用角色
- 前置条件
- 触发条件
- 主流程
- 异常流程
- 业务规则
- 状态及事件记录
- 输出结果
- 验收标准

核心功能包括认养激活、任务生成、任务分配、接单转派、执行提交、规则预检查、审核退回、天气延期、权益履约、指标比较、Bad Case 复盘和导出。

## 7. 规则与场景边界

V0/V1 只允许在分配、提醒、提交模板、规则预检查、审核优先级和天气延期规则上产生差异；两者必须共享种子、场景、配置和基础世界，不能人为改变村民能力或外部随机结果。

成稿覆盖八个场景：正常、认养高峰、人手不足、连续降雨、低提交质量、偏远区域负载、审核积压和采收高峰。

异常分支至少覆盖拒单、接单超时、重新分配、提交逾期、审核退回、规则检查不通过、暴雨延期、审核积压、权益延迟、库存不足、退款和争议。

## 8. 指标与结论设计

PRD 描述 13 项模拟指标及其产品含义，包括接单率、按时提交率、首次与最终审核通过率、重新分配率、逾期率、平均接单与审核时间、审核退回率、权益按时履约率、异常发现率、分配公平性和运营人工介入次数。

所有指标必须给出分子、分母、单位、排除项和零分母显示方式。比率差异使用百分点，非比率指标使用绝对差和安全变化率。

结论只允许四类：

- 模拟结果建议采用 V1
- 模拟结果暂不支持升级
- 存在场景退化，需要继续调整
- 样本不足，暂不形成结论

## 9. 作品集呈现原则

- 先讲业务问题与产品决策，再讲功能模块。
- 使用 Mermaid 流程图、状态图和表格帮助阅读，不加入装饰性图表。
- 技术架构只用于证明确定性、数据隔离和可追溯性，不展开代码细节。
- 明确区分“模拟验证结论”和“真实运营结论”。
- 产品经理复盘必须说明关键取舍、没有解决的问题、验证局限和下一步真实试点方案。
- 不虚构真实用户数量、真实收益、真实效率提升或真实运营结果。

## 10. 成稿交付标准

- 输出格式：Markdown。
- 目标路径：`docs/认养一棵树多场景规则测评系统PRD.md`。
- 章节完整，无占位符或未定义口径。
- 业务流程、状态机、V0/V1 规则、八个场景、13 项指标与验收标准保持一致。
- 全文持续使用“模拟”限定词，并在开头、指标章节和结论章节展示真实性声明。
- 文档可独立阅读，不要求读者先阅读技术交付文档。
