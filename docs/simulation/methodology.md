# 规则模拟方法与复现

> 项目处于 Demo 与业务验证设计阶段。模拟运营数据，不代表真实业务结果。

## 确定性方法

默认世界使用种子 `20260713`、30 天、100 笔认养、100 棵树、20 名村民、3 名审核员，每单 3—5 个任务。实体 ID 与决策类型组成带键随机子流，预先生成天气、出勤、接单倾向、耗时、质量、审核和物流扰动。规范化世界生成 `worldHash`，V0/V1 分别深拷贝运行，策略分支不会消耗或改变其他外生随机数。

回归种子固定为 `20260713`—`20260717`。相同配置与种子必须得到字节级相同的世界、事件和结果；不同种子应产生不同世界。成对比较同时校验种子、场景、完整配置和 `worldHash`，任一不一致都拒绝归因。

`durationDays` 是严格观察窗口。窗口外事件不进入结果，尚未到期的任务不进入到期类指标；审核员日容量形成确定性队列，场景压力只通过运行快照中的可审计参数生效。

## 八个场景

| 场景                   | 压力注入                                  |
| ---------------------- | ----------------------------------------- |
| NORMAL                 | 默认世界                                  |
| ADOPTION_PEAK          | 认养需求 1.8 倍，库存溢出形成 Bad Case    |
| STAFF_SHORTAGE         | 30% 村民与 1 名审核员不可用               |
| CONTINUOUS_RAIN        | 固定 5 天暴雨；仅影响与天气窗口相交的任务 |
| LOW_SUBMISSION_QUALITY | 提交质量扰动增加                          |
| REMOTE_ZONE_LOAD       | 远区任务占比增加                          |
| REVIEW_BACKLOG         | 审核队列处理时长和积压压力增加            |
| HARVEST_PEAK           | 收获及权益任务占比增加                    |

## 策略边界

V0 使用出勤和日容量约束下的名单轮转、12 小时接单等待、人工重派、一次截止提醒、自由提交和 FIFO 人工审核。V1 使用技能/距离/负荷/可靠性/可用性加权匹配、2 小时自动重派（最多 3 人）、24/6 小时提醒、结构化提交、基于规则的模拟凭证检查、天气延期和风险优先审核。

“基于规则的模拟凭证检查”只检查照片数量、清晰度、重复风险、字段完整度、树号、执行时间和描述长度；不调用大模型或真实图像审核。失败凭证和修正重提是不同、不可变的提交记录。

## 复现与限制

执行 `pnpm simulation:regression` 可复现 40 组成对结果；`pnpm simulation:export` 生成完整证据包。数据库不可用不阻断模拟，但文件/内存降级数据不适合作为长期审计存储。模拟只验证流程、规则和统计实现，不能证明真实用户行为、村民效率、运营收益或因果效果；上线策略仍需真实试点、监控和伦理/隐私复核。
