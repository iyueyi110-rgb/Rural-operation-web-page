## 🔴 P0 — 村民门户返回按钮缺少 fallback

### Fix 3: 村民门户返回按钮改用 BackButton 组件

**根因**: `villager/layout.tsx` 中返回按钮直接调用 `router.back()`，无 fallback。用户通过书签或直接 URL 进入子页面时点击返回无任何反应。

**涉及文件**: `apps/web/src/app/[locale]/villager/layout.tsx`

**修改方案**:

将当前原始 `<button onClick={() => router.back()}>` 替换为使用 `BackButton` 组件，并设置 fallback 到 dashboard：

```tsx
// 在文件顶部已有 import 中，确认存在 ArrowLeft（已有）

// 新增 import
import { BackButton } from "@web/components/back-button"

// 在 return 中，将以下代码：
// {!isDashboard ? (
//   <div className="sticky top-0 z-30 ...">
//     <button onClick={() => router.back()} ...>
//       <ArrowLeft ... />
//       {t("nav.back")}
//     </button>
//   </div>
// ) : null}

// 替换为：
{!isDashboard ? (
  <div className="sticky top-0 z-30 flex h-12 items-center border-b border-stone bg-white/90 px-4 backdrop-blur-sm">
    <BackButton
      className="flex cursor-pointer items-center gap-2 border-none bg-transparent text-sm font-bold text-moss transition hover:text-ink focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-moss"
      fallbackHref={`/${locale}/villager/dashboard`}
      label={t("nav.back")}
      icon={<ArrowLeft aria-hidden="true" className="h-4 w-4" />}
    />
  </div>
) : null}
```

**注意事项**:
- `BackButton` 是 `"use client"` 组件，`VillagerLayout` 已经是 `"use client"`，可以直接使用
- `locale` 变量在组件中已存在（`useLocale()`）
- fallback 到 dashboard 而非 login，因为 login 页面不显示返回按钮
- 移除不再需要的 `useRouter` import（如果其他位置还在用则保留）

## 🔴 P0 — 安全漏洞修复

### Fix 4: OTP 改用密码学安全随机数

**根因**: 游客和村民的 OTP 验证码使用 `Math.random()` 生成，可被预测。

**涉及文件**:
- `apps/web/src/app/api/v1/auth/request-sms/route.ts`
- `apps/web/src/app/api/v1/villager-auth/request-otp/route.ts`

**修改方案**:

将 `Math.random()` 替换为 `crypto.randomInt(100000, 999999)`：

```ts
// 修改前（示例）:
// const otp = String(Math.floor(Math.random() * 900000) + 100000)

// 修改后:
import { randomInt } from "node:crypto"
const otp = String(randomInt(100000, 999999))
```

**注意事项**:
- `randomInt` 来自 `node:crypto`（Node.js 内置）
- 返回值是 number，需要 `String()` 转换
- 两个文件都需要修改

### Fix 5: OTP 接口添加频率限制

**根因**: 发送验证码接口无任何限流保护，可被暴力枚举。

**涉及文件**: 同 Fix 4 的两个文件

**修改方案**:

在每个 POST handler 开头添加 Redis 限流检查：

```ts
import { checkRateLimit, getRateLimitKey, rateLimitResponse } from "@web/lib/rate-limit"

// 在 POST 函数体最开头（mobile 校验之前）添加：
const rateLimit = await checkRateLimit(getRateLimitKey(request, "otp-request"), 3, 60)
if (!rateLimit.allowed) return rateLimitResponse(request, rateLimit.resetAt)
```

**注意事项**:
- 限流必须在手机号校验**之前**执行，防止枚举攻击
- 同一 IP 每 60 秒最多 3 次请求
- 如果 Redis 不可用，`checkRateLimit` 会放行作为 fallback
- 需要确认 `@web/lib/rate-limit` 路径是否正确（如实际路径不同需调整 import）

## 🟡 P1 — 安全加固

### Fix 6: Cron Secret 从 URL Query 迁移到 Header

**根因**: `POST /api/v1/cron/daily-report` 从 URL query string 读取 secret，会被服务器日志记录。

**涉及文件**:
- `apps/web/src/app/api/v1/cron/daily-report/route.ts`
- `start.sh` / `start.ps1` / `start-mac.command`（curl 调用方）

**修改方案**:

1. 修改 `route.ts` 中的 POST handler：
```ts
// 修改前:
// const url = new URL(request.url)
// const secret = url.searchParams.get("secret")

// 修改后:
const secret = request.headers.get("Authorization")?.replace("Bearer ", "")
```

2. 修改启动脚本中的 curl 命令：
```bash
# 修改前:
# curl -X POST "http://localhost:3000/api/v1/cron/daily-report?secret=$CRON_SECRET"

# 修改后:
curl -X POST "http://localhost:3000/api/v1/cron/daily-report" \
  -H "Authorization: Bearer $CRON_SECRET"
```

**注意事项**:
- 三个启动脚本（`.sh`、`.ps1`、`.command`）都需要修改
- 同时检查是否还有别的 cron 端点使用相同模式

### Fix 7: 村民 Token 添加 JWT 签名

**根因**: `apps/web/src/lib/villager-auth.ts` 中 token 仅为 Base64 编码，可任意伪造村民身份。

**修改方案**:

```ts
// 修改 createVillagerToken():
import { SignJWT } from "jose"
import { jwtSecret } from "@web/lib/auth-jwt"

export async function createVillagerToken(villagerId: string) {
  return new SignJWT({ villagerId })
    .setProtectedHeader({ alg: "HS256" })
    .setIssuedAt()
    .setExpirationTime("7d")
    .sign(jwtSecret())
}

// 修改 getVillagerIdFromToken():
import { jwtVerify } from "jose"

export async function getVillagerIdFromToken(request: Request): Promise<string | null> {
  const header = request.headers.get("X-Villager-Token")
  if (!header) return null
  try {
    const { payload } = await jwtVerify(header, jwtSecret())
    return (payload.villagerId as string) ?? null
  } catch {
    return null
  }
}
```

**注意事项**:
- `jose` 库已在项目依赖中（`packages/utils` 或 `apps/web`）
- `getVillagerIdFromToken` 从同步变为异步，所有调用方需要 `await`
- 调用方至少包括 `requireVillagerRecipient` 和 `requireVillagerOrAdmin`（在 `auth-guards.ts` 或类似文件中）
- 村民端前端 `villager-auth-client.ts` 只存储/发送 token 不解析，不受影响
- **这是破坏性变更**——需要全面测试村民登录/鉴权流程
